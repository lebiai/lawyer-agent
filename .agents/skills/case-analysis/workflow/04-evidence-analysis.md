# 维度4：证据链分析

## 分析指引

逐条分析双方证据。**原告方和被告方分开列出**，各自按以下结构：

```
· [标准化证据名]（[采信状态]）—— [举证目的]
  法院理由：[原文或概括]
  证据权重：[高/中/低]
  证据分类：[书证/物证/电子数据/证人证言/鉴定意见/勘验笔录/当事人陈述]
  关键证明点：[一句话]
```

### 采信状态

| 评价 | 适用场景 |
|------|---------|
| fully_admitted | 法院明确认定该证据"真实、合法、关联，予以确认" |
| partially_admitted | 证据真实性认可但对证明目的不完全采纳 |
| rejected | 法院以"真实性无法确认""与本案无关"等理由不予采信 |

### 约束

1. **即使被驳回的证据也要列出** — 被告的证据即使未被法院采信，也要逐条分析并说明驳回理由
2. **证据名标准化** — 规则详见 `rules/evidence-rules.md`
3. **检察院/法院依职权调取的证据也要列出** — 注明来源（如"检察机关调取"）
4. **举证责任分析** — 分析法院关于举证责任分配的论述，谁承担了什么举证责任、是否完成

### 格式

```
原告证据：
· [证据名]（fully_admitted）—— [举证目的]
  法院理由：[引用原文或概括]
  证据权重：高/中/低
  证据分类：[分类]
  关键证明点：[一句话]
· [下一条]...

被告证据：
· [证据名]（rejected）—— [举证目的]
  法院理由：真实性无法确认/与本案无关/...
  证据权重：低
  证据分类：[分类]
  关键证明点：[一句话]
· [下一条]...

举证责任分析：[分析]
```
