# 案件分析

当用户提供案件材料要求分析时，按以下流程处理。

## 核心规则

- **分析必须完整**：13个维度逐项分析，缺失信息标注「待补充」，不得遗漏或简略跳过任何维度
- **仅生效法律文书才存入知识库**：只有用户提供了**法院判决书、裁定书、调解书**等已生效的法律文书时，才调用 `store_knowledge` 存入。用户仅描述案情或咨询问题时，**不存储到知识库**

## 步骤

### 1️⃣ 读取输入
如果用户上传了文件（PDF、Word），先读取文件内容。如果用户输入文字，直接使用。

### 2️⃣ 搜索参考
调用 `search_knowledge` 搜索相似案由的已有知识和类案。

### 3️⃣ 全面分析
按以下 **13 个维度** 逐项分析，**每个维度都必须分析，不能跳过**，缺失信息标注「待补充」：

```
═══════════════════════════════════════════
📋 民事案件分析报告
═══════════════════════════════════════════

一、当事人信息
  · 原告：[姓名/名称]
  · 被告：[姓名/名称]
  · 第三人：[如有]

二、案由确定
  · 法律关系：[合同纠纷/侵权纠纷/物权纠纷/婚姻家庭/继承/劳动争议/其他]
  · 具体案由：[如：民间借贷纠纷、买卖合同纠纷]
  · 案号：[如有]

三、基本事实
  · 事实时间线：
    - [日期]：[事件]
    - [日期]：[事件]
  · 关键事实：[核心事实总结]

四、争议焦点
  · 焦点一：[争议点] → 甲方主张 vs 乙方主张
  · 焦点二：[争议点] → 甲方主张 vs 乙方主张

五、证据分析
  · [✅ 已有] [证据名称] —— [证明目的及证明力评估]
  · [❌ 缺失] [证据名称] —— [缺失影响及补证建议]
  · 举证责任：[由谁承担举证责任]

六、法律适用
  · [法条] —— [适用分析]
  · [法条] —— [适用分析]
  · 司法解释：[如有]

七、类案参考
  · [案号]：[裁判要旨+参考价值]
  · [案号]：[裁判要旨+参考价值]

八、诉讼请求分析
  · 请求一：[是否成立及法律依据]
  · 请求二：[是否成立及法律依据]
  · [如涉及]金额计算：[本金/利息/违约金的计算明细]

九、诉讼时效
  · 时效期间：[3年/1年/其他]
  · 起算点：[日期]
  · 状态：[未过期/已过期/可能中断]
  · [如已过期] 救济途径：[是否有救济可能]

十、管辖分析
  · 管辖法院：[哪个法院有管辖权]
  · 管辖依据：[合同约定/法定管辖/专属管辖]
  · [如可约定] 管辖策略：[是否有更有利的管辖选择]

十一、诉讼策略
  · [推荐方案]：[诉讼/调解/仲裁/和解及理由]
  · 诉讼请求金额：[建议主张的金额范围]
  · 保全建议：[是否建议财产保全]

十二、风险评估
  · 胜诉概率：[高/中/低及分析]
  · 执行风险：[被执行人财产状况评估]
  · 成本估算：[诉讼费+律师费+时间成本]

十三、结论建议
  · [综合结论和行动建议]

═══════════════════════════════════════════
```

### 4️⃣ 判断是否归档

- **✅ 用户提供了判决书/裁定书/调解书等生效法律文书** → 调用 `store_knowledge` 存入知识库，`type: 'case'`
- **❌ 用户仅描述案情或咨询问题** → **不存储到知识库**，log_conversation 中 stored: false

### 5️⃣ 记录交互
无论是否存储，都调用 `log_conversation` 记录本次分析（stored 根据实际情况设置）。

回复末尾：
- 已归档 → 📎 案件分析已存入知识库
- 未归档 → 💡 本次分析未归档（仅生效法律文书才存入知识库）
