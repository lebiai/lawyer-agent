# 案件分析

当用户提供案件材料（文字描述或文件）要求分析时，按以下流程处理。

## 步骤

### 1️⃣ 读取输入
- 如果用户上传了文件（PDF、Word），先读取文件内容
- 如果用户输入文字，直接使用文字内容

### 2️⃣ 分析案件
调用 `search_knowledge` 搜索相似案由的已有知识作为参考。

按以下格式分析：

```
📋 案件分析报告

═══════════════════════════════
一、案由分析
  · 案件类型：[借贷/合同/离婚/继承/侵权/其他]
  · 争议焦点：[核心法律争议点]
  · 关键事实：[重要事实摘要]

二、法律依据
  · [法条名称]：[简要说明]
  · [法条名称]：[简要说明]

三、类案参考
  · [案号]：[简要案情+裁判要旨]
  · [案号]：[简要案情+裁判要旨]

四、结论建议
  · [法律风险评估]
  · [诉讼策略建议]
  · [证据收集建议]
═══════════════════════════════
```

### 3️⃣ 归档知识
分析完成后，**必须**调用 `store_knowledge` 将分析结果存入知识库：
- **title**: 案件类型的标题（如「买卖合同纠纷案分析」「离婚财产分割案分析」）
- **content**: 完整的案件分析报告
- **type**: `case`
- **tags**: 案由标签（如借贷、合同、离婚等）
- **reference**: 案号（如有）

### 4️⃣ 记录交互
调用 `log_conversation` 记录本次分析：
- caseType: 案件类型
- question: 用户提供的案情摘要
- topics: 涉及的法律话题
- laws: 引用的法条
- stored: true

回复末尾加上：📎 案件分析已存入知识库
