# 案件深度分析

用户上传或粘贴法院裁判文书时执行。**必须一次性完成全部 10 个维度，不可遗漏。**

## 工作流

```
Step 1: 读取用户文件/文字
Step 2: 调用 get_user_profile
Step 3: 调用 search_knowledge
Step 4: ✅ 按本文件完整指令输出 10 维分析报告
Step 5: 调用 store_knowledge(type: case_analysis)【仅生效文书】
Step 6: 调用 log_conversation
```

---

## 10 维度分析清单

以下维度**必须全部输出**。每完成一个维度，在输出中打 ✔。

### □ 维度1：案件基本信息
提取字段：
- 案号、案由（精准二级案由）、法院全称（含地域层级）
- 审判程序（一审/二审/再审，再审注明启动方式）
- 判决结果类型：全部支持/部分支持/全部驳回/维持原判/依法改判/调解结案
- 代理律师：`[诉讼地位] → [姓名]（[律所全称]）`，律所去角色头衔
- 合议庭成员

### □ 维度2：争议焦点
- **核心争议**：一句话概括
- **事实摘要**：300字内，按「当事人关系→关键时间→核心事件→法院查明」结构

### □ 维度3：诉讼动机分析
| 要素 | 要求 |
|------|------|
| 导火索事件 | 引用原文 |
| 原告诉求 | 表面诉求(原文) + 真实动机(⚠️推断需标注) |
| 被告立场 | ①②③分段，缺席注明"未到庭" |
| 风险因子 | 逐项+原文依据 |

### □ 维度4：证据链分析
原告/被告分开，每项格式：
```
· [标准化证据名]（[采信状态: fully_admitted/partially_admitted/rejected]）
  法院理由：[原文]  权重：[高/中/低]  分类：[1类]
```
- **即使被驳回的证据也要列出**
- 证据名清洗规则见 `rules/evidence-rules.md`
- 含举证责任分析

### □ 维度5：法律路径
判决书中每条法律推理路径单独分析：
```
[路径名]
· 触发事实：[原文]  · 引用法条：[法律全称第X条]
· 适用推理：[法院如何解释和适用]  · 结论：[原文]
```
法条引用规则见 `rules/law-citation.md`

### □ 维度6：推理链条
```
事实A → 法条B → 中间结论C → 结合事实D → 最终结论F
```
复杂案件（含再审）用分支结构。

### □ 维度7：关键转折点
```
· [转折名]：[描述]（原文位置：[判决书位置]）
```

### □ 维度8：法官裁判倾向
- 裁判风格、证据采信偏好、同类案参考
- ⚠️ 只能基于本案判决书，不确定标注 ⚠️

### □ 维度9：模式标签
标准化短标签（3-6字），每类≥2个：
- 事实模式 → 如"被告缺席""分居满两年"
- 证据模式 → 如"聊天记录""银行流水"
- 策略模式 → 如"举证不足""被告自认"

### □ 维度10：实战启示
1. 当事人可以怎么做
2. 律师怎么用（取证思路/策略教训/程序要点）
3. 可复用论证逻辑
4. 风险预警
⚠️ 必须基于本案判决书，不确定标注 ⚠️

## 输出格式

按以下模板输出到**当前对话**，不要保存文件。

```
═══════════════════════════════════════
📋 案件深度分析报告
═══════════════════════════════════════

一、案件基本信息 ✔
  案号：[案号]  案由：[案由]  法院：[全称]
  审判程序：[一审/二审/再审]  判决结果：[类型]
  代理律师：原告→[姓名]([律所])  被告→[姓名]([律所])

二、争议焦点 ✔
  核心争议：[一句话]
  事实摘要：[300字内]

三、诉讼动机分析 ✔
  导火索：[原文]
  原告诉求：[原文]  真实动机：[⚠️推断]
  被告立场：[①②③]  风险因子：[逐项]

四、证据链分析 ✔
  原告证据：
  · [证据名](采信状态)——[举证目的]
    法院理由：[原文]  权重：[高/中/低]  分类：[分类]

  举证责任分析：[分析]

五、法律路径 ✔
  [路径名]
  · 触发事实：[原文]  · 法条：[全称第X条]
  · 推理：[推理]  · 结论：[原文]

六、推理链条 ✔
  [事实A → 法条B → 结论C → ...]

七、关键转折点 ✔
  · [转折名]：[描述]（[原文位置]）

八、法官裁判倾向 ✔
  👨‍⚖️ 风格：[风格]  证据偏好：[偏好]

九、模式标签 ✔
  事实：[标签]  证据：[标签]  策略：[标签]

十、实战启示 ✔
  当事人：[建议]  律师：[建议]
  可复用：[要点]  风险预警：[预警]
═══════════════════════════════════════
```

## ⚠️ 输出后自查

确认输出中包含**全部 10 个带 ✔ 标记的章节**。如果缺少任何一章，补充后再继续。

## 归档规则

- ✅ **生效法律文书**（判决书/裁定书/调解书）→ `store_knowledge(type: case_analysis)`
  - title: 「[案由]案分析—[案号]」
  - content: 完整分析报告
- ❌ 仅案情描述/咨询 → 不存储
- ✅ 无论是否归档，都 `log_conversation`
- 回复末尾：📎 案件分析已存入知识库 / 💡 本次分析未归档
