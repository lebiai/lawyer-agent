# 输出模板

分析报告按以下 10 章结构直接输出到当前对话，不需要保存为文件。

```
═══════════════════════════════════════
📋 案件深度分析报告
═══════════════════════════════════════

一、案件基本信息
  案号：[案号]
  案由：[案由]
  法院：[法院全称]
  审判程序：[一审/二审/再审]
  判决结果类型：[全部支持/部分支持/全部驳回/维持原判/依法改判/调解结案]
  代理律师：
    原告方 → [姓名]（[律所]）
    被告方 → [姓名]（[律所]）

二、争议焦点
  核心争议：[一句话概括]
  事实摘要：[300字以内]

三、诉讼动机分析
  导火索事件：[原文标注]
  原告诉求：[原文]
  原告真实动机：[⚠️ 推断/原文]
  被告立场：[① ② ③]
  风险因子：[逐项列出]

四、证据链分析
  原告证据：
  · [证据名]（[采信状态]）—— [举证目的]
    法院理由：[原文]
    证据权重：[高/中/低]
  被告证据：
  · [同上格式]
  举证责任分析：[分析]

五、法律路径
  [路径名称]
  · 触发事实：[原文]
  · 引用法条：[全称第X条]
  · 适用推理：[推理]
  · 结论：[原文]

六、推理链条
  [箭头串联：事实A → 法条B → 结论C → ...]

七、关键转折点
  · [转折名称]：[描述]（原文位置）

八、法官裁判倾向
  裁判风格：[描述]
  证据采信偏好：[描述]
  同类案参考：[描述]

九、模式标签
  事实模式：[标签]、[标签]
  证据模式：[标签]、[标签]
  策略模式：[标签]、[标签]

十、实战启示
  当事人可以怎么做：[基于本案]
  律师可以怎么用：[策略/取证/程序要点]
  可复用要点：[论证逻辑]
  风险预警：[趋势/注意]

═══════════════════════════════════════
```

每条分析结论后标注原文位置，不确定性标注 ⚠️，详见 rules/quality-control.md。
