# 质量约束与反例

## 输出质量要求

### 1. 每个结论必须可溯

每个分析结论后需标注在判决书原文中的位置。格式：

```
· 法院认定该协议有效（判决书第2页"本院认为"段）
· 银行转账记录被法院采信（判决书第5页"证据认定"部分）
```

### 2. 不确定性标注

如果你的分析有一部分是**推断**而非原文直接表述，用 ⚠️ 标记：

```
· 原告真实动机：⚠️ 争夺房屋增值收益（原文未明确表述，从诉请推断）
```

### 3. 绝对性承诺禁止

不得输出以下类型表述：
- "一定能胜诉"
- "法院会支持"
- "保证拿回"
- "必然败诉"

应替换为：
- "从历史判例看，相似事实法院倾向于支持原告/被告"
- "本案事实与 XX 案类似，可以参考"

## 不合格输出的反例

### 反例1：证据名未清洗
```
❌ "801房房产证原件（2023年4月由原告提交）" — 合格证据名应为"房产证"
❌ "彭某手写的《保证书》《悔过书》" — 合格应为"保证书、悔过书"
```

### 反例2：法条引用未写全称
```
❌ "民法典第1079条" — 应为"中华人民共和国民法典第一千零七十九条"
❌ "根据婚姻法解释" — 应为"中华人民共和国民法典婚姻家庭编的解释(一)第X条"
```

### 反例3：编造心理状态
```
❌ "原告当时很愤怒" — 除非判决书明确写"原告情绪激动"
❌ "被告感到冤枉" — 除非判决书明确写了
```

### 反例4：模糊引用
```
❌ "根据相关法律规定" — 必须写出具体法条
❌ "同上"、"同前" — 不允许相对引用
```

### 反例5：无依据结论
```
❌ "这个案子上诉一定会改判" — 绝对性承诺
❌ "法院明显判错了" — 主观评价，非基于原文分析
```

### 反例6：只分析被采信的证据
```
❌ 被告证据：无（全部被驳回）
✅ 被告证据：
   · 刘某某名下银行流水（rejected）—— 证明资金来源于申某
     法院理由：不足以推翻一、二审法院认定
```
被驳回的证据也必须逐条列出并说明驳回理由。

### 反例7：遗漏程序法路径
```
❌ 只分析实体法路径
✅ 同时分析：
   · 实体法：民法典第1079条 → 感情破裂认定
   · 程序法：民诉法第147条 → 缺席判决程序依据
```
